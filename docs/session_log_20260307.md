## 2. 중복제거 알고리즘 설명

### 요청

> 중복제거 알고리즘을 자세히 설명

### 분석 대상

`src/preprocessor/dedup.py` (46줄)

### 핵심 내용

- **알고리즘:** `difflib.SequenceMatcher` (Ratcliff/Obershelp)
- **유사도 공식:** `2.0 * M / T` (M=매칭 문자수, T=총 문자수)
- **임계값:** 80% 이상이면 중복 판정
- **메타데이터:** 중복 뉴스의 `coverage_count`와 `coverage_sources`를 원본에 병합
- **시간 복잡도:** O(N*U), 300건 기준 ~38,000회 비교, <1초

### 처리 예시

```
입력 4건 → 중복제거 → 출력 2건
A "삼성전자 Q4 매출 50% 증가" (naver)        → 유지 (coverage_count: 3)
B "삼성전자, Q4 매출 전년대비 50% 증가" (yahoo)  → A에 병합
C "SK하이닉스 HBM 수주 사상 최대" (google)     → 유지 (coverage_count: 1)
D "삼성전자 4분기 매출 50% 급증 전망" (rss)     → A에 병합
```

## 6. 뉴스 랭킹 알고리즘 분석 (주가 영향 제외)

### 요청

> 주가의 영향을 제외하고 종목별로 가장 뉴스를 결정하는 알고리즘을 설명

### 분석 대상

`src/ranking/engine.py` — `_fallback_news_ranking()` 메서드

### 주가 제외 시 3가지 순수 뉴스 품질 지표

**1. Coverage Score (보도 범위, 0~25점)**

```python
coverage_score = min(25, coverage_count * 8)
```

| coverage_count | 점수 |
|---------------|------|
| 1 | 8 |
| 2 | 16 |
| 3 | 24 |
| 4+ | 25 (상한) |

**2. Keyword Score (키워드 중요도, 0~25점)**

5단계 티어, 최고 등급 우선 매칭:

| 티어 | 점수 | 키워드 |
|------|------|--------|
| 1 | 25 | M&A, 인수합병, 상장폐지, 거래정지, bankruptcy |
| 2 | 20 | 실적발표, 영업이익, 매출, earnings, revenue |
| 3 | 15 | 공시, 유상증자, 자사주, regulation, tariff |
| 4 | 10 | 금리, 환율, Fed, CPI, GDP |
| 5 | 5 | 반도체, AI, 2차전지, 바이오 |
| 미매칭 | 2 | 일반 뉴스 |

**3. Source Credibility (소스 신뢰도, 0~25점)**

```
dart: 25, yahoo: 18, rss: 18, naver: 15,
google_news: 14, reddit: 8, twitter: 8, naver_discussion: 6
```

**4. Market Relevance (주가 제외 시, 0~17점)**

- 종목코드 매핑됨: +12점
- 카테고리가 M&A/실적/공시/정책·규제: +5점

### 주가 제외 시 점수 범위

```
총점 = Coverage(0~25) + Keyword(2~25) + Source(6~25) + Relevance(0~17) = 8~92점
```

## 8. 본문 기반 중요도 판단 방법

### 요청

> 글의 순수내용만 가지고 중요도를 파악할 수 있을까

### 제안된 4가지 방법

**1. 수치 임팩트 추출 (규칙 기반)**

뉴스 본문의 구체적 숫자(금액, 변동률, 수량, 극단 표현)를 정규식으로 추출:

```
"삼성전자, 10조원 규모 투자 결정" → 금액 10조 → 높은 점수
"반도체 업종 좋은 흐름" → 구체적 수치 없음 → 낮은 점수
```

**2. 사건 유형 분류 (행위 동사 기반)**

| 중요도 | 동사 |
|--------|------|
| 최고 | 결정, 확정, 승인, 체결 (확정된 사실) |
| 높음 | 추진, 검토, 계획 (진행 중) |
| 중간 | 전망, 예상, 관측 (예측) |
| 낮음 | 가능성, 루머, 소문 (불확실) |

"삼성전자 인수합병 **결정**"(25점) vs "삼성전자 인수합병 **가능성**"(5점)

**3. 정보 밀도 측정 (구체성 vs 모호성)**

고유명사, 숫자, 날짜, 종목코드 등 구체적 팩트의 밀도를 측정.

**4. LLM 직접 평가 (가장 정확, 비용 높음)**

본문을 LM Studio에 보내서 직접 중요도 판단. 토큰 소모 큼.

### 제안된 조합

```
Content Score (0~25) = 수치 임팩트(0~10) + 사건 확정도(0~10) + 정보 밀도(0~5)
```

## 9. ML 기반 중요도 예측 구조 설계

### 요청

> 머신러닝 기법으로 test 글의 중요도를 입력하고 그것을 기반으로 train의 중요도를 예측하는 구조를 만들 수 없을까

### 설계된 구조

**워크플로우:**
1. 사람이 뉴스 200~300건에 중요도 점수(0~100) 수동 라벨링
2. 텍스트 특징 추출 (TF-IDF + 수작업 특징 15개)
3. LightGBM 회귀 모델 학습
4. 새 뉴스 입력 시 중요도 자동 예측

**Feature 구성:**
- 텍스트 벡터: TF-IDF 3,000차원 또는 Sentence Transformer 768차원
- 수작업 특징: 수치 관련(4개), 확정도(2개), 구조적(4개), 극단 표현(3개), 메타(2개)

**피드백 루프:**
- 사람 라벨 300건 → 초기 모델
- `news_impact` 테이블의 실제 주가 변동 → 자동 라벨 생성
- `auto_label = |change_pct_1d| * 10`
- 모델 재학습 → 사람 라벨 + 자동 라벨 결합

## 10. Self-Supervised 금융 AI 알고리즘 설계

### 요청

> 뉴스 중요도를 자동으로 만드는 Self-Supervised 금융 AI 알고리즘을 만들 수 있을까

### 생성 파일

`docs/self_supervised_algorithm.md`

### 핵심 개념

사람 라벨링 없이 **시장 반응 자체를 정답**으로 사용:

- Signal 1: **비정상 수익률** (시장 평균 대비 초과 수익률)
- Signal 2: **거래량 이상** (평소 대비 거래량 배수)
- Signal 3: **변동 지속성** (1일 후에도 영향 유지되는지)

### 자동 라벨 공식

```
종목 뉴스:
  market_signal = price_score(0~50) + volume_score(0~30) + persistence(0~20)
  label = market_signal * 0.7 + coverage * 0.3

거시 뉴스:
  market_signal = 시장 전체 변동률 기반
  label = market_signal * 0.5 + coverage * 0.5
```

### 자동 개선 흐름

| 시점 | 데이터 | 모델 상태 |
|------|--------|----------|
| Day 1~3 | 수집만 진행 | 규칙 기반 100% |
| Day 4~7 | 자동 라벨 50~100건 | ML 20% + 규칙 80% |
| Day 14 | 200건+ | ML 40% + 규칙 60% |
| Day 30 | 500건+ | ML 100% |
| Day 90 | 1500건+ | 계절성 패턴 학습 |
| Day 365 | 전체 사이클 | 상승장/하락장별 패턴 |

### 기존 스코어 구성 변경

```
기존: Coverage(25) + Keyword(25) + Source(25) + Relevance(25) = 100점
변경: Coverage(20) + Keyword(20) + Source(20) + Relevance(20) + Content(20) = 100점
                                                                 ↑ ML 예측
```

## 핵심 인사이트 (세션 통합)

### 현재 시스템의 구조적 한계

현재 랭킹 시스템은 **뉴스 본문의 내용**을 거의 분석하지 않고, 메타데이터(소스, 보도량, 키워드 유무)에 의존. "무엇을 말하는가"보다 "누가, 몇 번 말했는가"로 판단하는 구조.

### 진화 경로

```
현재 상태                    단기 개선                     장기 목표
규칙 기반 4항목 스코어링  →  본문 분석 추가 (정규식)    →  Self-Supervised ML
(Coverage, Keyword,       (수치 임팩트, 사건 확정도,     (시장 반응 자동 라벨,
 Source, Relevance)        정보 밀도)                    TF-IDF + LightGBM)
```

### 실행 우선순위

1. **즉시 가능:** Reddit score, 토론방 views를 기존 스코어링에 반영
2. **단기:** 수치 임팩트 + 사건 확정도 규칙 기반 Content Score 추가
3. **중기:** Self-Supervised 모델 구현 (데이터 3일 축적 후 자동 학습 시작)
4. **장기:** Sentence Transformer 임베딩으로 정확도 향상
