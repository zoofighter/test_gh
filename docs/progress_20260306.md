# 주식 랭킹 뉴스 시스템 - 작업 진행 현황

작성일: 2026-03-06

## 프로젝트 개요

주식 관련 뉴스를 다양한 소스에서 수집하고, 전처리 후 DB에 저장하여
이후 랭킹 엔진 및 분석 모듈에서 활용할 수 있도록 하는 시스템.

참조 구현체: `0224_a/` (완성된 31개 모듈, 2,911줄)

## 완료된 작업

### 1단계: 뉴스 수집 모듈 (2026-03-05)

8개 소스에서 뉴스를 수집하는 모듈 구현.

**생성 파일:**

| 파일 | 설명 |
|------|------|
| `src/collectors/base.py` | NewsItem 데이터클래스 + BaseCollector 추상 클래스 |

| `src/collectors/yahoo.py` | Yahoo Finance (yfinance, 미국 종목) |
| `src/collectors/rss.py` | RSS 피드 (한국경제, 매일경제) |
| `src/collectors/google_news.py` | Google News RSS (토픽 + 검색) |
| `src/collectors/reddit.py` | Reddit JSON API (stocks, wallstreetbets, investing) |
| `src/collectors/twitter.py` | Twitter via Nitter (현재 비활성) |
| `src/collectors/naver_discussion.py` | 네이버 종목토론방 (현재 비활성) |
| `config.yaml` | 수집기별 설정, watchlist, DB 경로 |
| `requirements.txt` | 의존성 패키지 목록 |
| `main.py` | 실행 진입점 (CLI) |

**NewsItem 필드:**
```
source, title, url, content, published_at, stock_code, stock_name, category, extra
```

**수집 결과 (테스트):**

- 총 297건 수집

### 2단계: DB 저장 모듈 (2026-03-05)

SQLite 기반 데이터 저장 및 조회 모듈 구현.

**생성 파일:**

| 파일 | 설명 |
|------|------|
| `src/database/models.py` | DB 스키마 정의 (8개 테이블) + init_db() |
| `src/database/repository.py` | Repository 클래스 (CRUD 전체) |

**DB 테이블 구조:**

| 테이블 | 용도 |
|--------|------|
| `stocks` | 종목 마스터 (code, name, market, sector) |
| `news` | 수집된 뉴스 (source, title, url, content, published_at, category, stock_code) |
| `stock_prices` | 주가 데이터 (stock_code, date, close, volume, change_pct) |
| `rankings` | 종목 랭킹 결과 |
| `ranking_news` | 랭킹-뉴스 매핑 (M:N) |
| `news_rankings` | 개별 뉴스 랭킹 결과 |
| `news_impact` | 뉴스 발행 후 주가 영향 추적 |
| `scheduler_log` | 스케줄러 실행 로그 |

**주요 기능:**
- URL 기반 중복 체크 (`news_exists`)
- 종목 자동 등록 (6자리 숫자 → KOSPI, 영문 → NYSE)
- 30일 지난 데이터 자동 정리 (`cleanup_old_data`)
- FK 제약 충족을 위한 `register_stocks()` → `save_to_db()` 순서 보장

**저장 결과 (테스트):**
- 269건 수집 → 250건 저장, 19건 중복 스킵
- 5개 미국 종목 자동 등록 (AAPL, TSLA, NVDA, MSFT, AMZN)

### 3단계: 전처리 모듈 (2026-03-06)

수집과 DB 저장 사이에 전처리 파이프라인 추가.

**파이프라인:**
```
collect_news() → deduplicate() → map_stocks() → categorize() → save_to_db()
```

**생성 파일:**

| 파일 | 설명 |
|------|------|
| `src/preprocessor/dedup.py` | 제목 유사도 기반 중복 제거 (threshold 80%) |
| `src/preprocessor/kr_stock_dict.py` | pykrx 기반 한국 전체 종목 사전 (캐시) |
| `src/preprocessor/stock_mapper.py` | 뉴스→종목코드 매핑 (DB + pykrx + 패턴) |
| `src/preprocessor/categorizer.py` | 키워드 기반 8개 카테고리 분류 |

**각 모듈 상세:**

**dedup.py** - 중복 제거
- difflib.SequenceMatcher로 제목 유사도 계산
- 유사도 80% 이상이면 중복 판정
- 원본에 coverage_count(보도 매체 수), coverage_sources(매체 목록) 메타데이터 추가
- 결과: 297건 → 257건 (40건 중복 제거)

**stock_mapper.py** - 종목 매핑
- 1순위: DB 종목명 + pykrx 사전 (긴 이름 우선 substring 매칭)
- 2순위: 6자리 숫자 패턴 (한국 종목코드)
- 3순위: 1-5자 대문자 영문 패턴 (미국 티커)
- pykrx 매칭 종목은 DB에 자동 등록
- 결과: 3건 종목 매핑

**kr_stock_dict.py** - 한국 종목 사전
- pykrx로 KOSPI + KOSDAQ 전체 종목 로드
- 모듈 레벨 캐시 (프로세스당 1회)
- 실패 시 빈 딕셔너리 반환 (에러 격리)

**categorizer.py** - 카테고리 분류
- 8개 카테고리: 실적, 공시, M&A, 신사업, 거시경제, 섹터이슈, 수급, 정책·규제
- 제목+내용에서 한국어/영어 키워드 매칭 횟수로 최적 카테고리 선택
- 이미 카테고리가 있는 항목(DART 공시 등)은 건너뜀
- 결과: 257건 중 122건 분류 완료

## 실행 방법

```bash
# 의존성 설치
pip install -r requirements.txt

# 전체 실행 (수집 + 전처리 + DB 저장)
python main.py

# 상세 로그 출력
python main.py -v

# DB 현황 조회만
python main.py --stats

# 수집만 (DB 저장 안 함)
python main.py --no-save

# 수집 건너뛰기
python main.py --no-collect
```
