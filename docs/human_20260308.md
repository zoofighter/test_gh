## 1. 주가 없이 순수하게 뉴스만으로 랭킹을 매기는 시스템
- 뉴스를 보고 사람이 직접 순위를 매김
- 그것을 토대로 랭킹을 매기는 시스템을 만듬
- 그리고 그 검증을 주가로 하고 싶지만 검증하기 어렵다.
- 그 검증을 사람이 직접 하는 것과 에이전트가 하는 것으로 나누어서 진행
- bert 같은 모델은 사용할 필요는 없을 것 같다.
- 많이 사용되는 키워드나 문맥을 파악해서 랭킹을 매기는 시스템을 만듬(참고사항)
- 클릭이 많은 뉴스를 어떻게 분류할지 노출 빈도가 많아서 그런 건지
- 사람이 판단한 뉴스 중에서 (의미가 있는 뉴스) 주가에 영향을 미친 뉴스를 찾아서 모델을 학습시키고 싶다.(즉 랭킹을 매기는 모델을 만들고 싶다.
)
- 지금까지 아이디어로 self_supervised 알고리즘 생성
- 로직이 만들어지면 그때 주가와 거래량으로 검증하고 싶다.

## 다음에 참조할 것

1. 네이버 기사 댓글 수 (가장 실현 가능)
2. 네이버 뉴스 검색 API (반응도 측정)
동일 키워드 검색 결과 수 = 해당 이슈의 보도 규모
3. Google Trends (이슈 관심도) 
특정 키워드의 시간대별 검색량 추이
4. 네이버 토론방에 하루 올라오는 글의 개수

5. LLM 직접 평가 (가장 정확, 비용 있음)
평가 기준:
- 확정된 사실인가 vs 추측/전망인가
- 구체적 수치(금액, 변동률)가 포함되어 있는가
- 특정 기업에 직접적인 영향이 있는가
- 시장 전체에 파급력이 있는가
JSON으로 응답: {"content_score": 점수, "reason": "근거"}
6. TF-IDF - Sentence Transformers
7. 검증에서 에이전트의 역할 
    - llm이 뉴스평가, ML 모델이 매긴점수를  검토
8. 스킬이 도움이 될까
9. 랭체인과 에이전트를 도입할 수 있는 방법
10. 스를 분리하는 기능 가령 실적과 예상 주가 관련, 산업 관련, 기얼 기술 관련 등등

## 에이전트를 도입할 수 있는 3가지 포인트
1. LLM 라벨링 에이전트 (llm_labeler.py)
현재 계획: LLM이 뉴스를 읽고 1~5점 부여

기본 방식 (지금과 동일):


프롬프트 → LLM → JSON 응답 → 파싱
에이전트 방식:


LLM이 뉴스를 받으면:
  → Tool 1: DB에서 같은 종목의 과거 뉴스 조회
  → Tool 2: 해당 종목의 최근 주가 추이 확인
  → Tool 3: 유사 뉴스의 기존 라벨 참조
  → 종합 판단 후 등급 부여
LangChain의 create_react_agent로 구현 가능. LLM이 스스로 필요한 정보를 도구로 조회하고 판단합니다.

2. 검증 에이전트 (plan의 Step 4 검증)
사용자 문서에 적힌 내용:

"LLM이 뉴스평가, ML 모델이 매긴 점수를 검토"

에이전트 방식:


검증 에이전트가 할 일:
  → Tool 1: ML 모델의 예측 점수 조회
  → Tool 2: 사람 라벨과 비교
  → Tool 3: 점수 차이가 큰 건 분석 (왜 다른지)
  → 검증 보고서 생성
3. 뉴스 분석 에이전트 (기존 랭킹 강화)
현재 NewsRankingEngine은 뉴스 배치를 한 번에 던지고 점수를 받습니다.

에이전트 방식:


뉴스 하나를 받으면:
  → Tool 1: 키워드 추출 (TF-IDF)
  → Tool 2: 유사 뉴스 검색 (DB)
  → Tool 3: 해당 기업 최근 공시 확인
  → 맥락을 종합하여 점수 산출
